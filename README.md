# DoD Projects

A portfolio of self-directed projects exploring C++ (with MFC), Java (with Swing), and REST APIs.

Each project is built end-to-end with documentation explaining design decisions, not just working code.

## Roadmap

### Phase 1: Foundations
Core language fluency in C++ and Java through a shared domain model — an aerospace parts inventory tool.

- **Project 1:** C++ CLI Inventory Tool
- **Project 2:** Java CLI Inventory Tool

### Phase 2: GUI Layer
Desktop frontends on the same domain model, using GUI frameworks.

- **Project 3:** MFC Desktop Work Order Tracker (C++)
- **Project 4:** Swing Desktop Work Order Tracker (Java)

### Phase 3: REST APIs
Moving from local persistence to a service-backed architecture.

- **Project 5:** Spring Boot REST Backend (Java)
- **Project 6:** MFC client refactored to consume REST API
- **Project 7:** Swing client refactored to consume REST API

### Phase 4: Capstone
- **Project 8:** Multi-Client Internal Tool — one backend, two desktop clients, full architecture documentation

## Tech stack

- **Languages:** C++17, Java 17
- **GUI frameworks:** MFC, Swing
- **Backend:** Spring Boot, PostgreSQL
- **Build systems:** CMake, Maven/Gradle
- **Testing:** Google Test, JUnit 5

## Status

Currently working through Phase 1. See individual project READMEs for progress and design notes.
